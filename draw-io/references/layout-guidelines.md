# 布局指南

## 1. 分组原则

- AWS Cloud 组作为最外层
- 按功能单位创建子组
- 组以横向排列为基本,按照数据流向配置

### 1.1. 组的层次结构

```text
AWS Cloud (最外层)
├── VPC
│   ├── Public Subnet
│   │   └── ALB, NAT Gateway 等
│   └── Private Subnet
│       └── ECS, RDS 等
├── S3
├── CloudWatch
└── 其他服务
```

## 2. 连接线规则

### 2.1. 线种的使用区分

| 流程类型 | 线种 | 用途 |
|-----------|------|------|
| Ingestion Flow | 破线 | 数据摄取 |
| Query Flow | 实线 | 查询・参照 |
| Control Flow | 点线 | 控制・管理 |

### 2.2. 箭头的方向

- 箭头遵循数据流动的方向
- 双向通信使用双向箭头

## 3. 配置原则

### 3.1. 从左到右的流程

```text
[数据源] → [处理] → [存储] → [分析/可视化]
```

### 3.2. 从上到下的流程 (替代方案)

```text
[用户/客户端]
        ↓
[负载均衡器]
        ↓
[应用程序]
        ↓
[数据库]
```

## 4. 确保可视性

- 标签配置在元素附近
- 调整配置以避免箭头交叉
- 相关元素分组并靠近配置
- 适当留白以提高可读性
